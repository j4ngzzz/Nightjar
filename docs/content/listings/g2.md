## Product Overview

*Paste into the "Product Description" or "About" field. Plain text preferred. Approx. 400–450 words.*

---

Nightjar is a verification pipeline for Python code. Where static analysis tools check for known-bad patterns, Nightjar checks whether code satisfies behavioral contracts — mathematical statements about what the code must always do. The difference matters: pattern-matching tools find known-bad code; Nightjar finds code that is semantically wrong even when it looks correct.

The pipeline runs six stages in sequence, cheapest first. Stage 0 checks syntax and dead constraints. Stage 1 scans dependencies for CVEs via pip-audit. Stage 2 validates schema invariants via Pydantic. Stage 2.5 runs negation proofs to confirm the spec itself is satisfiable. Stage 3 runs property-based tests using Hypothesis, generating hundreds of edge-case inputs automatically. Stage 4 runs formal symbolic verification using CrossHair and Dafny. When Stage 4 fails, a CEGIS (Counterexample-Guided Inductive Synthesis) loop extracts the exact failing input and feeds it back into an LLM repair cycle.

Teams start without writing any specs. `nightjar scan <file>` extracts invariant contracts from existing Python source using AST analysis. `nightjar audit <package>` produces an A–F report card for any PyPI dependency. The full spec-driven workflow becomes available when teams are ready to write `.card.md` contract files.

Nightjar's verification approach was validated by scanning 34 production Python packages. The pipeline found 74 confirmed bugs with zero false positives across that set. Critical findings include authentication bypasses in JWT and OAuth libraries, a Unicode ENS address hijacking vector in web3.py, and shared mutable session state in Google's Agent Development Kit. Fourteen packages were verified clean, including datasette (1,129 functions), rich, and hypothesis itself. The full scan results and reproduction scripts are publicly available.

For compliance-sensitive teams, Nightjar generates artifacts directly relevant to the EU Cyber Resilience Act (CRA, effective September 2026): a JSON verification report (`verify.json`), a sealed dependency manifest with SHA-256 hashes (`deps.lock`), and a read-only audit branch of all generated code. These outputs provide machine-readable evidence of security due diligence without requiring manual documentation.

Nightjar integrates into existing workflows: GitHub Actions (SARIF output for PR annotations), pytest (`--nightjar` flag), pre-commit hooks, VS Code diagnostics, and any MCP-compatible AI coding tool via a three-tool MCP server. Dafny is optional — without it, the pipeline falls back to CrossHair and Hypothesis and still produces a calibrated confidence score.

## Differentiators

*Paste into the "Key Differentiators," "Features," or "What makes your product unique?" field. Use the bullet format the form provides.*

---

**Finds semantic bugs, not just known patterns**
Property-based testing and symbolic execution generate edge-case inputs automatically. The pipeline found JWT expiry bypasses (`if exp` is falsy when `exp=None`), Unicode normalization flaws, and logic errors that no pattern-based SAST rule could express. These bugs require executing code against generated inputs — not matching source text against rules.

**Six-stage pipeline in a single command**
`nightjar verify` chains dependency CVE scanning, schema validation, negation proofs, Hypothesis property tests, and formal symbolic proof. Each stage feeds failure context to the next. The pipeline short-circuits on the first failure to minimize CI time.

**Spec as the permanent artifact**
In Nightjar's model, the `.card.md` spec is the source of truth. Code is regenerated from the spec on every build and is never manually edited. This creates a permanent, git-versioned record of intent that is machine-verifiable, not just human-readable documentation.

**Zero-friction entry for existing codebases**
`nightjar scan` bootstraps specs from existing Python code. `nightjar verify --fast` skips formal proofs and runs in under 10 minutes on large codebases. Teams do not need to adopt the full spec-driven workflow to get value from Stages 1–3 immediately.

**EU CRA compliance artifacts built in**
`nightjar ship` generates a compliance certificate alongside the build artifact. The verification report, dependency manifest, and audit branch together constitute a machine-readable due-diligence trail for CRA compliance without manual documentation work.

**CEGIS repair loop**
When verification fails, Nightjar does not just report the failure. It extracts the concrete counterexample, formats it as structured feedback, and passes it back to an LLM repair cycle. This closes the gap between "found a bug" and "here is the fix."

**Self-verifying**
Nightjar runs its own verification pipeline on its own codebase in CI. Every spec has a proof. If Nightjar's own code violates one of its own contracts, CI fails. This is a verifiable claim — the CI badge on the repository reflects the current state.
