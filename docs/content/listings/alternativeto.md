Nightjar is a verification pipeline for Python that checks whether code actually does what it claims to do — for every possible input, not just the ones you tested.

The core idea: you write a `.card.md` spec describing what a function must always guarantee. Nightjar runs six automated stages — syntax preflight, dependency CVE scanning, schema validation, negation proofs, property-based testing (Hypothesis), and formal proof (CrossHair + Dafny) — and reports whether the code satisfies the spec or produces a counterexample showing exactly how it can fail.

For existing codebases, `nightjar scan` extracts invariant contracts from Python source without any spec writing. `nightjar audit <package>` scores any PyPI package A–F. No spec required to start.

To validate the approach, Nightjar scanned 34 popular Python packages and found 74 confirmed bugs with zero false positives. A sample: `web3.py` silently resolves fullwidth Unicode ENS names to different Ethereum addresses (ETH address hijacking vector). `fastmcp` accepts JWT tokens from 1970 and tokens with no expiry field. `openai-agents` lets any agent invoke any other agent's tools across trust boundaries. `pydantic v2`'s `model_copy(update={...})` bypasses all field validators. All findings have standalone reproduction scripts.

Fourteen packages were clean. The pattern held: hand-maintained projects with active test suites passed. AI-generated codebases did not.

Nightjar outputs SARIF for GitHub Code Scanning, VS Code diagnostics, and a compliance artifact for EU Cyber Resilience Act (September 2026 deadline). The Dafny formal prover is optional — without it, Nightjar falls back to CrossHair and Hypothesis and still produces a mathematical confidence score.

Every line of Nightjar itself was generated by AI. Every line has a spec. Every spec has a proof. It dogfoods its own pipeline in CI.
